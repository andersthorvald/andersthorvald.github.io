Database Failover Runbook

📅 2026-05-25 📁 Runbooks database failover runbook

Pre-check

  • Verify primary DB is unreachable from app servers
  • Check monitoring alerts for DB health

Step 1: Identify Primary Failure

# Check current primary status
psql -h <primary-host> -c "SELECT pg_is_in_recovery();"

Step 2: Promote Replica

# On replica server
pg_ctl promote -D /var/lib/postgresql/data

Step 3: Update Connection String

# Update application config
DATABASE_URL=postgresql://appuser:password@<new-primary>:5432/mydb

Verification

  • Test connection from app servers
  • Monitor replication lag on remaining replicas
  • Verify read replicas are pointing to new primary
